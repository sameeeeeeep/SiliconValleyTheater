# CLAUDE.md — SiliconValley Theater

## What This Is

A macOS menu bar app that watches Claude Code sessions and generates live comedy commentary voiced by TV characters. The goal is ELI5: a non-programmer following along should understand what's happening through the characters' banter about the work.

## Build & Run

```bash
make build                    # → build/SiliconValleyTheater.app
open build/SiliconValleyTheater.app
```

No Xcode project — uses `swiftc` directly via Makefile.

## What Actually Works (verified)

```
JSONL events → SessionWatcher (kqueue, <1ms)
  → EventSummarizer (extracts Claude's assistant text)
  → buildTemplateDialogue (reformats into character turns)
  → TTS synthesis (pipelined: line N+1 while N plays)
  → Playback through floating widget

Side channels that work:
  → Term explainers from theater.md (keyword-triggered, play after main dialogue)
  → Static fillers from FillerLibrary.swift (context-matched by tags)
  → Voice cache (WAV on disk, 90%+ hit rate after first session)
  → Cold opens from theater.md (play on session start)
```

## What's Built But Unreliable

- **ELI5 rewriter** (`simplifyDialogue()`): sends template lines to 3B model for simplification. Fails often — model drops formatting or produces gibberish. Falls back to raw template lines.
- **preStripTechnical()**: removes file names, code syntax, camelCase from text before sending to model. Works mechanically but only matters when the rewriter succeeds.
- **theater.md jargon map**: project-specific term replacements (kqueue → "file change detector"). Wired into pre-stripper. Same dependency on rewriter.
- **Sentiment-aware filler tags**: build-fail vs build-success tagging exists. Untested in practice because fillers mostly play before any events arrive.

## Architecture

### Core Loop (TheaterEngine.swift, ~800 lines)
1. SessionWatcher delivers events via AsyncStream
2. Events buffer for 3 seconds
3. EventSummarizer extracts user ask + Claude explanation + file changes
4. buildTemplateDialogue reformats into character turns
5. (Attempted) ELI5 rewriter simplifies via 3B model — usually fails, falls back to raw template
6. TTS synthesizes each line (pipelined)
7. Widget displays current speaker + subtitle

### Filler System (DynamicFillerPool.swift)
- **Static fillers**: hardcoded per theme in FillerLibrary.swift, sentiment-tagged
- **theater.md fillers**: loaded on session pin, inserted at front of pool
- **LLM-generated fillers**: replenished during idle periods
- **Context matching**: `extractTags()` derives tags from recent events, `consumeNext()` picks best match
- **Neutral-only fallback**: situational fillers (build-fail, test-fail) don't play without matching context

### theater.md System (TheaterContext.swift)
- Parsed from `.claude/theater.md` in each project
- Contains: cold opens, fillers, term explainers, voice examples, cast mapping, jargon map
- Generated by `/create-theater` skill
- Loads lazily on session pin or first event — not available at app startup

## File Layout

- `Sources/Engine/TheaterEngine.swift` — central orchestrator, event batching, playback pipeline
- `Sources/Engine/DynamicFillerPool.swift` — filler pool, tag matching, replenishment
- `Sources/Services/DialogueGenerator.swift` — template builder, ELI5 rewriter, LLM prompt construction
- `Sources/Models/TheaterContext.swift` — theater.md parser, RAG matching, jargon map
- `Sources/Models/FillerLibrary.swift` — hardcoded static fillers per theme
- `Sources/Models/CharacterTheme.swift` — 9 character themes with personalities and catchphrases
- `Sources/Models/Config.swift` — settings, per-session theme map, LLM/TTS provider selection
- `Sources/Views/WidgetView.swift` — floating widget, character tiles, subtitles (yellow for fillers)
- `Sources/SiliconValleyApp.swift` — menu bar app entry point

## Common Tasks

- **Fix dialogue quality**: `DialogueGenerator.swift` — template builder and ELI5 rewriter prompt
- **Fix filler selection**: `DynamicFillerPool.swift` — `extractTags()` and sentiment tag lists
- **Fix theater.md loading**: `TheaterEngine.swift` — `selectRoom()`, `handleEvent()` lazy load
- **Add a new theme**: `CharacterTheme.swift` + matching fillers in `FillerLibrary.swift`
- **Generate theater.md**: `/create-theater` skill

## LLM Providers

- **Ollama** (default): local Qwen 2.5 3B — `ollama pull qwen2.5:3b`
- **Groq** (optional): cloud Llama 3.3 70B — free API key from console.groq.com

## TTS Providers

- **Pocket TTS** (default): local voice cloning via Python sidecar in `TTSSidecar/`
- **Kokoro**: fast local TTS
- **Fish Audio / Cartesia**: cloud alternatives

## Known Issues

- ELI5 rewriter fails most of the time on 3B — raw Claude text plays instead
- theater.md not available at startup (loads on session pin), so first fillers/intros are from hardcoded library
- 3B model can't reliably stay in character — the pre-written theater.md content sounds better than anything it generates
- Voice cache grows forever (~/.siliconvalley/voice_cache)
